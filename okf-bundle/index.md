# Estatly Spec Knowledge Bundle (OKF)

The canonical home for Estatly's authored spec knowledge as an [Open Knowledge Format](https://github.com/GoogleCloudPlatform/knowledge-catalog/blob/main/okf/SPEC.md) bundle — one concept per markdown file, YAML frontmatter, ordinary markdown links. The original requirements brief (`__init.md`, retired 2026-07-20) is fully absorbed here; scope lives in [Product Overview](specifications/product-overview.md). See [authoring conventions](_conventions.md).

# Concept types

* [specifications](specifications/index.md) — Narrative feature and domain specs derived from the requirements brief.
* [architecture](architecture/index.md) — Backend and platform architecture concepts.
* [api](api/index.md) — API Contracts — one `api/<surface>/` per client surface: REST narrative + `openapi.yaml`.

# Meta

* [Authoring conventions](_conventions.md) — `type` vocabulary, frontmatter fields, linking rules, conformance check.
* [log](log.md) — change history.
